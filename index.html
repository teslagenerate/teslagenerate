### итак мой первый проэкт на этом ресурсе

Этот автоматический генератор страницы-это самый простой способ создавать красивые страницы для всех ваших проектов. Автор ваш контент страницы с помощью github ароматный уценки, выберите шаблон, созданный дизайнером, и опубликовать. После того как страница создана, вы можете проверить новый `гр-страницы филиала на местном уровне. Если вы используете github для рабочего стола, просто синхронизируйте ваше хранилище и вы увидите новую ветку.

# Шаблоны Конструктор#

Мы обработали некоторые красивый шаблоны для вас, чтобы использовать. Идите вперед и нажмите кнопку 'продолжить макетов', чтобы просмотреть их. Вы можете легко вернуться к редактировать страницу перед публикацией. После публикации Ваша страница, вы можете пересмотреть генератора страницу и перейти на другую тему. Содержание страницы будут сохранены.

Создавать страницы вручную

Если вы предпочитаете не использовать автоматический генератор, нажимаем ветку под названием `гр-страниц на свой репозиторий создать страницу вручную. В дополнение к поддержке обычных HTML-содержимое, страницы github поддержки Джекил, простой, блог знают статический генератор сайтов. Джекил позволяет легко создавать веб-узла колонтитулах, без необходимости копировать их на каждой странице. Он также предлагает интеллектуальные блог поддержки и других дополнительных шаблонов функций.

Авторы и участники

Вы можете @имя пользователя github, чтобы создать ссылку на свой профиль. В результате `<элемент> будет ссылка на профиль в github автора. Например: в 2007 году, Крис Wanstrath (@defunkt), PJ Хиеттом (@pjhyett), и том Престон-Вернер (@mojombo) основана на github.

# Поддержка# или контакт

Возникли проблемы со страницами? Проверьте наши документация или обратитесь в службу поддержки, и мы поможем Вам разобраться.
